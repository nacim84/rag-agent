# Commands - Workflows Opérationnels

## Vue d'ensemble

Ce dossier contient des "Playbooks" ou guides de procédures pas à pas pour les agents IA et les développeurs. Contrairement aux `skills` (qui sont techniques) ou aux `agents` (qui sont des rôles), les `commands` sont des **séquences d'actions** pour accomplir des tâches spécifiques.

## Index des Commandes

1. **[EPCT](./epct.md)** - Workflow standard de développement (Evaluate, Plan, Code, Test)
2. **[DEBUG](./debug.md)** - Méthodologie de résolution de bugs et Root Cause Analysis
3. **[REFACTOR](./refactor.md)** - Guide pour le refactoring de code sans régression
4. **[DOCUMENT](./document.md)** - Workflow pour la mise à jour de la documentation technique

## Comment utiliser ces commandes

Lorsqu'une tâche est confiée, l'agent doit :
1. Identifier la commande appropriée.
2. Suivre rigoureusement chaque étape définie dans le fichier `.md`.
3. Valider chaque étape avant de passer à la suivante.
